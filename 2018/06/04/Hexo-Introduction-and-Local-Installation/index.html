Hexo Introduction and Local Installation

What is Hexo?

Hexo is a fast, simple, and high-efficient blog framework. MarkDown is supported by Hexo. Hexo can generates static websites based on nice themes. And it only needs one command to deploy to GitHub.

Hexo Installation

Hexo can be installed within several minutes. Before that, make sure that these following things are set up already:

  • Node.js
  • Git

If you have these already, then use npm(What is npm?) to install Hexo:

If not, following these step:

  • Git
  • Node.js

If you failed to install Git or Node.js, here are two tips to help you:

  • Git Installation on Windows
  • Node.js

Hexo Configuration

Initialise your target <folder>.
Here, I named my target folder as “blog”.

After initialization, your project folder will look like this:

|- _config.yml
|- package.json
|- scaffolds
|- source
|  |-  _drafts
|  |_  _posts
|_ themes

_config.yml

File for configuration of the website, most of the settings are set up here.

package.json

Application data. EJS, Stylus, and MarkDown renderers are installed by defalut here. You can delete these afterwards if you want.

scaffolds

When new post is created, Hexo bases new file on the scaffolds.

source

All website content will be put here. Hexo will ignore hidden files, files or folders name start with  _ ( or dash sign).

themes

Hexo generates static websites by combining web contents and theme.
