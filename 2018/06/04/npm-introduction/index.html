npm Introduction

What is npm?

npm is known as node package manager. npm makes it easier for JavaScript developers to share codes they have created for particular problems. Also, other developers can reuse these codes on their own applications.
Once you use npm, it is easier to know whether these codes updated or not.

These re-usable codes are called as packages, or some times modules. A package is only a registry, which has a file or several files inside. Meanwhile, a package.json file is included in this package, which has some metadata about this package.
A typical application, like a website, may needs a lots of packages. Packages normally are small but are aimed at specific problems. So, you can use many packages to solve all sorts of different questions in your own project, which finally forms your project. Package also makes it possible to reuse these codes in your different projects.
You can find and install packages on your own project through npm websites.

What npm refers?

Normally, npm refers to three things:

  • npm website:
    The website you can find and download the package you want.
  • registry:
    a big database of information about this package.
  • npm client:
    When developers want to publish or update their codes to registry, they can do this through the npm client installed on their own computers. After published, other developers are available to download and install these packages through the entry of registry.

Thus, npm is a good way to share, reuse, and manage diverse version of codes.
