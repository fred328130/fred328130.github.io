MarkDown introduction

What is MarkDown?

Markdown is a lightweight markup language with plain text formatting syntax. It is designed so that it can be converted to HTML and many other formats using a tool by the same name. Markdown is often used to format readme files, for writing messages in online discussion forums, and to create rich text using a plain text editor.

What is markup language?

Markup language is a type of computer literal coding, which combines the text and its corresponding related information to render the document structure and data handling details. The information regarding to the text, including the document structure and the express information, combines with the text together and is marked by markup language.

What is lightweight markup language?

Lightweight markup language (LML) is type of literal coding, which applies simple syntax to describe simple formats. As a kind of markup language, LML has easy-understanding grammar, and is convenient to be edited. Its original style is close to natural language. The reason why it is called lightweight is because of the comparison between LML and other complicated markup languages, like RTF, HTML, and TeX.

Difference between MarkDown and HTML

MarkDown is mainly used to write readme files. So it doesn’t have too much complicated markup tags, comparing to HTML. In somehow, HTML is like a Microsoft Word, which has all sorts of sophistictated functions to allow you adjust the font size, font color, line height, and etc. Contrarily, MarkDown is like a text editor, which has limited and simple functions to write plain text files.

MarkDown example

  1. Italic
    A pair of underscore( _ italic _ ) is used to italize the corresponding content.

    Content written in original MarkDown:

    _ italic content _

    Corresponding effect:

    italic content

  2. Bold
    A pair of two asterisks ( **bold**) is used to phrase the text.

    Content written in original MarkDown:

    ** bold content **

    Corresponding effect:

    bold Content

  3. Headers

    There are six types of headers, in decreasing sizes, used in MarkDown. Phrase prefaced with a hash mark( # ) is set to be the header. However, header one and header six need to be utilized carefully.

    Content written in original MarkDown:

    # This is header one.
    ## This is header two.
    ### This is header three.
    #### This is header four.
    ##### This is header five.
    ###### This is header six.

    Corresponding effect:

    This is header one.

    This is header two.

    This is header three.

    This is header four.

    This is header five.
    This is header six.

  4. Links
    There are two types of link in MarkDown. The first link is called inline link. In order to create this, the link text should be wrapped within two brackets ( [ ] ), and then link to wrapped in parenthesis( ( ) ).

    Content written in original MarkDown:

    [Visit GitHub!](wwww.github.com)

    Corresponding effect:

    Visit GitHub!

    The other type of link is called reference link. As the name implies, the link is actually a reference to another place in the document. An advantage of the reference link style is that multiple links to the same place only need to be updated once. For example, if we decide to make all the same links go somewhere else, we only have to change the single reference link.
    Reference links don’t appear in the rendered Markdown. You define them by providing the same tag name wrapped in brackets, followed by a colon, followed by the link.

    Content written in original MarkDown:

    [Visit Youtube][Youtube link]
    Youtube link:www.youtube.com

    Corresponding effect:

    Visit Youtube

  1. Images
    The way to create images is similar to links. There are two types of images. One is the inline image link. To create an inline image link, enter an exclamation point ( ! ), wrap the alt text in brackets ( [ ] ), and then wrap the link in parenthesis ( ( ) ). (Alt text is a phrase or sentence that describes the image for the visually impaired.)

    Content written in original MarkDown:

    ![Bannekat](https://octodex.github.com/images/bannekat.png)

    Corresponding effect:

    The other type of image link is called reference image link. For a reference image, it is the same pattern as a reference link.

    Content written in original MarkDown:

    [Bannekat][Ban Cat]
    [Ban Cat]:https://octodex.github.com/images/bannekat.png

    Corresponding effect:

    Bannekat

  1. Blockquotes
    A blockquote is a sentence or paragraph that’s been specially formatted to draw attention to the reader. To create a block quote, all you have to do is preface a line with the “greater than” caret (>).

    Content written in original MarkDown:

    >This is a sentence needs to be quoted.

    Corresponding effect:

    This is a sentence needs to be quoted.

    Notice that even blank lines must contain the caret character. This ensures that the entire blockquote is grouped together. Also, Blockquotes can contain other Markdown elements, such as italics, images, or links.

  2. Lists
    There are two types of lists in the known universe: unordered and ordered. That’s a fancy way of saying that there are lists with bullet points, and lists with numbers.
    To create an unordered list, you’ll want to preface each item in the list with an asterisk ( * ). Each list item also gets its own line.

    Content written in original MarkDown:

    * Milk
    * Eggs
    * Salmon
    * Butter

    Corresponding effect:

    • Milk
    • Eggs
    • Salmon
    • Butter

    An ordered list is prefaced with numbers, instead of asterisks.

    Content written in original MarkDown:

    1. Milk
    2. Eggs
    3. Salmon
    4. Butter

    Corresponding effect:

    1. Milk
    2. Eggs
    3. Salmon
    4. Butter

    If one list needs to be nested inside another list, MarkDown supports this by indenting each asterisk one space more than the preceding item.

    Content written in original MarkDown:

    * A
     * A.1
     * A.2
     * A.3
    * B
     * B.1
     * B.2
     * B.3
      * B.3.1
      * B.3.2

    Corresponding effect:

    • A
      • A.1
      • A.2
      • A.3
    • B
      • B.1
      • B.2
      • B.3
        • B.3.1
        • B.3.2

    Normally, it is not recommended to write more than three levels in lists.
    Furtheremore, if some extra content requires to be added within the bullet list. To create this sort of text, your paragraph must start on a line all by itself underneath the bullet point, and it must be indented by at least one space.

    Content written in original MarkDown:

    1. Milk
      Milk is good.
    2. Eggs
      Eggs are good.

    Corresponding effect:

    1. Milk
      Milk is good.
    2. Eggs
      Eggs are good.

  3. Paragraphs
    In MarkDown, writing a new paragraph cannot be achieved by only typing cores content in their own line. In such case, MarkDown will only render this as a straight line. However, if you forcefully insert a new line, you will break the togetherness. This is what we called as hard break. This issue can be solved by inserting two spaces after each line.

Content written in original MarkDown:

This is paragraph A.**
This is paragraph B.**

Corresponding effect:

This is paragraph A.
This is paragraph B.

The asterisk(*) is used to represented as a space here.
